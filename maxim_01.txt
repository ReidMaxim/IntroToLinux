## LINUX_OVERVIEW

-- FULL_NOTES (ORIGINAL)
Linux is a free, open-source operating system. It was created in 1991 by Linus Torvalds, who was a college student at the time.

Linus wrote the Linux kernel and made it public, inviting others to build on it. Linux by itself is just the kernel. Most people refer to things like Ubuntu, Arch, and Kali as “Linux,” but these are technically distributions of Linux. These distributions package together the Linux kernel, system tools, and in some cases a desktop environment.

From what I can gather, Linus made the Linux kernel to be a free, open-source, Unix-like kernel. Unix was created by Bell Labs in the 1970s and was described as powerful and elegant. It was used by professionals in universities, research labs, telecommunications, and engineering firms.

Despite this, Unix was expensive; its source code was restricted, and it was not accessible to the normal or home user. In college, Linus was using MINIX, which was a Unix-like operating system that was also limited by licensing and design choices. Linus wanted to create a free kernel that worked like Unix but without the limits of licenses or cost.

One of the ways Linux differs from other operating systems is that it focuses on control and customization, allowing users to create an operating system that works for their needs, compared to Windows, which tries to make things simple but at the cost of control and customization.

## KERNEL

-- FULL_NOTES (ORIGINAL)
A kernel is low-level software that manages hardware and provides system resources and services to all programs.

The kernel uses device drivers to understand and communicate with each piece of hardware. Every program running on the system is a process, and the kernel creates and destroys these processes.

The kernel also manages memory, controls file systems, handles networking, and enforces security.

============================================================

## LINUX_KERNEL_LANGUAGE

-- FULL_NOTES (ORIGINAL)
The Linux kernel was mostly written in C, with small sections written in assembly.

C is powerful and is described as being as “close to the hardware” as a programming language can be. The translation from C to hardware instructions is direct and transparent. C also produces fast and predictable code, and it can be compiled for many different CPU architectures.

This explains how Linux can run on so many different systems, such as x86 and ARM architectures.

## ENTERPRISE_AND_GAMING_LINUX_DISTRIBUTIONS

-- FULL_NOTES (ORIGINAL)
Enterprise Linux distributions

Some examples of enterprise Linux distributions include:

Red Hat Enterprise Linux (RHEL)

An industry standard for large corporations, especially in banking and finance. It focuses on long-term stability and conservative updates.

Ubuntu Server

Free to use, with optional paid support. Popular in cloud and DevOps environments and offers long-term support (LTS) releases.

SUSE Linux Enterprise

Popular in Europe. It features live patching and high-availability clustering, which allows multiple devices or nodes to work together to ensure little to no downtime for services or data access.

These distributions are considered enterprise distributions because they are tailored for businesses, large organizations, data centers, and mission-critical operations.

Linux gaming distributions

Linux distributions made for gaming usually include GPU drivers, performance tuning, and easy setup for popular services like Steam.

SteamOS

Built by Valve and optimized for the Steam Deck.

Garuda Linux

An Arch-based distribution that features aggressive tuning and is very performance-focused.

Pop!_OS

An NVIDIA-friendly distribution that can auto-detect or allow users to select an OS version based on their graphics drivers.

## SOFTWARE_LICENSES_AND_LINUX

-- FULL_NOTES (ORIGINAL)
What is the GPL?

GPL = General Public License

In simple terms, you are free to:

use the software

study it

modify it

share it

Anything that is modified must also be open source.

This protects the user’s freedom to tinker and modify. The Linux kernel itself is GPL-licensed.

============================================================

## MIT_LICENSE

-- FULL_NOTES (ORIGINAL)
What is the MIT License?

The MIT License is a permissive license.

Users can:

use it

copy it

modify it

sell software using it

They can also keep their changes closed source and do not have to share modifications.

It includes a “no warranty” rule: if the code breaks your stuff, you cannot sue the author. Users must still credit the original creator.

============================================================

## GPL_VS_MIT

-- FULL_NOTES (ORIGINAL)
Major Differences Between GPL and MIT

With the GPL, you cannot keep changes secret or use it in closed-source software.

With the MIT License, you do not have to share modifications, making it more vendor-friendly.

How Big Vendors Use Linux

============================================================

## MBR

-- FULL_NOTES (ORIGINAL)
MBR (Master Boot Record)

Partition information stored in one small spot at the start of the disk

Supports up to 4 partitions

Works on old computers and legacy BIOS

Maximum disk size: ~2 TB

Single point of failure

============================================================

## GPT

-- FULL_NOTES (ORIGINAL)
GPT (GUID Partition Table)

Stores partition information in multiple locations

Helps prevent a single point of failure from corruption

Has a backup copy of the partition table

Supports very large disks (far beyond MBR limits)

Supports many partitions (up to 128 in most cases)

Designed for newer hardware and UEFI

Rule:

If a system uses UEFI, use GPT

If a system uses legacy BIOS, use MBR

============================================================

## UEFI

-- FULL_NOTES (ORIGINAL)
UEFI (Unified Extensible Firmware Interface)

UEFI is the modern replacement for BIOS. It is the first software that runs when you power on a computer.

UEFI:

Wakes up the hardware

Checks the CPU, RAM, disks, keyboard, mouse, etc.

Looks for boot instructions

Launches the operating system

UEFI can be described as:

“The OS launcher that lives below the operating system.”

Why BIOS Was Replaced

Slow startup

Limited disk support

Lack of security features

BIOS vs UEFI

BIOS

UEFI

Old

New

Very limited

Flexible

MBR

GPT

No security

Secure Boot

Tiny boot code

Real boot programs

## FILE_SYSTEM_COMPARISONS

-- FULL_NOTES (ORIGINAL)
File System Comparisons

EXT4

Strengths

Very stable

Well-tested

Fast for everyday use

Low overhead

Works everywhere on Linux

Easy to recover if something breaks

Weaknesses

No native snapshots

Limited modern features

No built-in checksumming

Best for:

Desktops

Laptops

Servers that value stability

Beginners who “just want it to work”

BTRFS (B-tree File System)

A modern Linux file system

Adds advanced features without extra tools

Strengths

Snapshots

Copy-on-write

Checksums for data and metadata

Easy rollback after bad updates

ZFS (Zettabyte File System)

File system and volume manager with RAID

Focused on data integrity and massive storage capacity

Strengths

End-to-end checksumming

Automatic data repair

Snapshots and clones

Designed for massive storage systems

Summary

EXT4: best for general use

BTRFS: best for advanced desktops

ZFS: best for servers and NAS

## VIRTUALBOX

-- FULL_NOTES (ORIGINAL)
Setup and guide for VirtualBox

[https://youtu.be/XvyQPsXY_oQ?si=d6SrnsW-qYuAeALY](https://youtu.be/XvyQPsXY_oQ?si=d6SrnsW-qYuAeALY)

[https://www.virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)

What is VirtualBox?

VirtualBox is a program that lets you run a complete computer inside your computer.

That “computer inside a computer” is called a virtual machine (VM).

Your real computer:

hardware

OS (Windows/macOS/Linux)

files

VirtualBox:

pretends to be hardware

creates fake CPU, RAM, disk, network

runs another OS safely inside a window

What problem VirtualBox solves

VirtualBox lets you:

try Linux without installing it

run multiple OSes at once

break things safely

learn OS concepts hands-on

test installers, partitions, encryption, etc.

If the VM breaks:

Close it. Delete it. Try again.

Your real system is untouched.

The mental model (this matters)

Think of it like this:

Host OS → your real OS (Windows/macOS/Linux)

VirtualBox → the pretend hardware factory

Guest OS → the OS inside the VM (Fedora, Mint, etc.)

Your computer

├─ Host OS

│   └─ VirtualBox

│       └─ Guest OS (Linux)

What VirtualBox actually emulates

Each VM gets:

virtual CPU cores

virtual RAM

a virtual hard drive (just a file)

a virtual network card

a virtual display

Linux thinks it’s running on real hardware — but it’s not.

## VIRTUALBOX_SETUP_AND_USAGE

-- FULL_NOTES (ORIGINAL)
How to use VirtualBox (step by step, detailed but safe)

Install VirtualBox

Download VirtualBox from the official site

Install it like any normal application

(Optional but recommended) install the Extension Pack

adds USB support

adds better device integration

You now have a VM manager, not a VM yet.

Get a Linux ISO

You’ll need:

a Linux ISO file (Fedora, openSUSE, Mint, etc.)

Example:

linuxmint-21.3-cinnamon-64bit.iso

This ISO is the “installer CD”.

Create a new Virtual Machine

In VirtualBox:

Click New

Name the VM (e.g., “Linux Mint Test”)

Choose:

Type: Linux

Version: Ubuntu (Mint-based) or Fedora, etc.

This just sets defaults — nothing permanent yet.

Assign RAM and CPU

RAM (memory)

Minimum: 2 GB

Recommended: 4 GB if you can spare it

CPU

1–2 cores is fine

Don’t assign more than half your real cores

Rule of thumb:

Leave enough resources for your host OS.

Create a virtual hard disk

This is a file on your real disk that pretends to be a hard drive.

Choices:

VDI (VirtualBox format) → fine

Dynamically allocated → grows as needed

Size:

Minimum: 20 GB

Comfortable: 30–40 GB

This is where Linux will live.

Attach the ISO (this is important)

Before starting the VM:

Open Settings

Go to Storage

Under “Controller IDE” or “Controller SATA”

Attach the Linux ISO file

This tells the VM:

“Boot from this installer.”

Start the VM

Click Start.

What happens:

VirtualBox pretends to power on a computer

The VM boots from the ISO

You see the Linux installer

From here on:

You install Linux exactly like a real computer

Install Linux inside the VM

Inside the installer:

choose language

choose keyboard

choose Erase disk (safe — it’s the virtual disk!)

create user

install

 Important:

“Erase disk” only erases the virtual disk file, not your real drive.

Reboot & remove ISO

After installation:

VM reboots

Linux starts from the virtual disk

If it boots back into the installer:

remove the ISO from Settings → Storage

 Install Guest Additions (highly recommended)

Guest Additions improve:

screen resolution

mouse integration

clipboard sharing

performance

Steps:

VM running

Menu → Devices → Insert Guest Additions CD

Run installer inside Linux

Reboot VM

Now the VM feels much smoother.

Common things people use VirtualBox for

Learning Linux

Testing installers

Practicing partitioning

Trying encryption (LUKS)

Running servers locally

School labs

Cybersecurity practice

What VirtualBox is NOT good for

Heavy gaming

GPU-intensive tasks

Real performance testing

Hardware driver testing

It’s a learning and testing tool, not a replacement for native installs.

One-sentence summary (lock this in )

VirtualBox is software that lets you safely run full operating systems inside your computer by emulating hardware, making it ideal for learning, testing, and experimenting with Linux.

## OBS

What is OBS?

OBS Studio (Open Broadcaster Software) is a free, open-source program for recording and streaming video.

People use OBS to:

record their screen

record gameplay

record lectures or demos

livestream to Twitch, YouTube, etc.

capture webcams and microphones together

It’s used by:

streamers

YouTubers

teachers

developers

students (screen recordings + voice)

The core idea (this is the mental model)

OBS works with three main concepts:

Scenes

A scene is a layout.

Think:

“What do I want visible right now?”

Examples:

Full screen + mic

Screen + webcam in corner

Slides only

Game capture

Sources

Sources are the things inside a scene.

Common sources:

Screen capture

Window capture

Game capture

Webcam

Microphone

Image

Text

Scenes are made of sources stacked together.

Output

Output is where the video goes:

a video file (recording)

a livestream (Twitch, YouTube)

The OBS interface (what you’re looking at)

When OBS opens, you’ll see:

Scenes (bottom left)

Sources (next to scenes)

Audio Mixer (mic & desktop sound levels)

Controls (Start Recording, Settings, Exit)

Nothing happens until you add sources.

## OBS_USAGE

-- FULL_NOTES (ORIGINAL)
How to use OBS (step by step, detailed but safe)

Install OBS

Download OBS Studio

Install it like any normal program

First launch may ask to run an auto-configuration wizard

This is safe to accept for beginners

Create your first scene

OBS usually starts with a scene called “Scene”.

You can:

keep it

or rename it (e.g., “Screen Recording”)

A scene is empty until you add sources.

Add a screen or window (most common use)

To record your full screen:

Click + under Sources

Choose Display Capture

Name it

Select your monitor

Click OK

To record one app only:

Click +

Choose Window Capture

Pick the app window

Now you’ll see your screen inside OBS.

Add your microphone 🎙

Usually OBS auto-detects your mic.

Check the Audio Mixer:

You should see green bars moving when you talk

If not:

Go to Settings → Audio

Select your microphone under Mic/Aux

Apply

(Optional) Add a webcam

Click + under Sources

Choose Video Capture Device

Select your webcam

Resize and drag it in the preview

Sources stack like layers:

top = front

bottom = back

Adjust audio levels (important)

In the Audio Mixer:

Your mic should peak in yellow, not red

Desktop audio should be audible but not overpowering

You can:

drag sliders

mute sources

add filters (noise suppression, gain)

Choose recording settings (quality control)

Go to Settings → Output → Recording.

Important options:

Recording format: MKV (recommended)

Encoder: Hardware (if available)

Quality: “High Quality” or “Indistinguishable”

Why MKV?

If OBS crashes, the file is not corrupted

You can remux to MP4 later

Choose where recordings save

Settings → Output → Recording Path

Pick a folder you’ll remember.

Start recording

Click Start Recording.

OBS now:

captures video

mixes audio

writes to disk in real time

Click Stop Recording when done.

That’s it — you just made a recording.

Streaming vs Recording (quick clarity)

Recording → saves video locally

Streaming → sends video to the internet live

Streaming requires:

a platform (Twitch, YouTube)

a stream key

more bandwidth

Recording is much simpler for beginners.

Common beginner mistakes (so you avoid them)

Forgetting to add a source → black screen

Recording at huge resolution → massive files

Mic too quiet or too loud

Not checking audio before recording

Recording desktop audio accidentally (echo)

What OBS is great at

Screen recording

Tutorials

Demos

Class projects

Streaming

Content creation

What OBS is not great at

Video editing

Heavy effects

Mobile recording

OBS records — editing comes later.

One-sentence summary (lock this in )

OBS is free software for recording and streaming video that works by arranging sources into scenes and outputting them as recordings or livestreams.
